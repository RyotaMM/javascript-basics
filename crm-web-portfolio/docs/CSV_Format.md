# 店舗データ CSV フォーマット仕様（データ提供者向け）

店舗巡回CRMに取り込むためのCSVの作り方です。**記入用テンプレート：`store_import_template.csv`** を使うと確実です。

---

## 列の一覧

| 列名（ヘッダー） | 必須 | 説明・例 |
|----------------|:---:|---------|
| 店舗コード | 任意 | 店舗を識別する固有コード（例: S001）。**あると更新が正確**になります（後述） |
| 店舗名 | **必須** | 例: 渋谷ラーメン道 |
| 住所 | **必須** | 都道府県から番地まで。例: 東京都渋谷区宇田川町1-1 |
| 店舗ジャンル | 任意 | 例: ラーメン屋 / 美容室 |
| 営業時間 | 任意 | 例: 11:00〜23:00、水曜定休 |
| 大エリア | 任意 | 例: 東京都 |
| 中エリア | 任意 | 例: 渋谷 |
| 小エリア | 任意 | 例: 宇田川町 |
| 現在の最新ステータス | 任意 | 未記入なら「未訪問」。下記の決まった名称で記載 |
| 最新の訪問メモ | 任意 | 自由記述 |
| 緯度 | 任意 | 例: 35.6615（後述） |
| 経度 | 任意 | 例: 139.6976（後述） |

> 1行目は必ずこのヘッダー行にしてください。列の順番は問いません。

---

## 記入ルール

### 文字コード
**UTF-8（BOM付き可）** または **Shift-JIS**。Excelで「CSV UTF-8」で保存すればOKです。

### ステータス（現在の最新ステータス）
次の**決まった名称**で記載してください（アプリのマスタと一致させます）:
`未訪問` / `再訪問` / `設置済み` / `お断り` / `NG`
- 未記入の場合は自動で「未訪問」になります。
- 一覧に無い名称は取り込み時に「未訪問」へ読み替え、警告が出ます（勝手に新ステータスは作りません）。

### エリア（大・中・小）
名前で記載すれば、システム側で自動的に階層（大→中→小）として登録されます。
- 表記ゆれ（例: 「渋谷」と「渋谷区」）は別エリア扱いになるため、**呼び方を統一**してください。

### 緯度・経度（任意）
地図にピンを表示するための座標です。
- **入っていれば**：取り込み後すぐ地図にピンが表示されます。
- **空欄でもOK**：その店舗は「位置未設定」となり、後から地図上でピンを置いて配置できます。
- ※ 住所からの自動座標変換は行いません。座標がある場合のみこの列に入れてください（無ければ空欄で構いません）。

---

## 更新の仕組み（重要）

同じCSVを再取り込みすると、**既存店舗は更新／無い店舗は新規追加**されます（重複登録されません）。

- **突合（同一店舗の判定）**: `店舗コード` があれば最優先。無ければ「`店舗名` + `住所`」で判定。
  → **店舗コードを付けておくと、店名や住所が多少変わっても同じ店として正しく更新**できます。
- **更新時に上書きするのは基本情報のみ**（店舗名・住所・ジャンル・営業時間・エリア）。
  現場担当者がアプリで入力した**ステータス・メモ・訪問記録はCSVで上書きしません**（現場の記録を保護）。

## サンプル（テンプレートの中身）

```
店舗コード,店舗名,住所,店舗ジャンル,営業時間,大エリア,中エリア,小エリア,現在の最新ステータス,最新の訪問メモ,緯度,経度
S001,渋谷ラーメン道,東京都渋谷区宇田川町1-1,ラーメン屋,11:00〜23:00,東京都,渋谷,宇田川町,未訪問,,35.6615,139.6976
S002,新宿イタリアン カフェ,東京都新宿区新宿3-1-2,イタリアン,11:30〜22:00,東京都,新宿,新宿3丁目,未訪問,,,
```

- S001：緯度経度あり → 取り込み後すぐ地図に出ます
- S002：緯度経度なし → 「位置未設定」で登録（後から地図で配置）

---

## まとめ（最低限これだけでOK）

- **必須は「店舗名」と「住所」だけ**。他は任意。
- 1行＝1店舗、1行目はヘッダー。
- できれば **店舗コード** と **緯度経度** も入れると、更新の正確さと地図表示がスムーズになります。
